RESTful Documentation

RESTful Services available under ADMIN authority authentication.


Spring Data REST Service with HAL Browser

note 1: Spring Security set to allow POST, PUT, DELETE at this entry point ONLY

note 2: This app uses a NoSQL Database. Some referential integrity check is done at app level but proceed with care.

  • /restful/api

Other RESTful Services available as GET-only methods:

  • /restful/fight/{userName}/{charName}/{locationName} - to get the full parameters for one fight, by user name, character name and location name
    notes: one monster is always picked randomly from location and the die is re-rolled for every cast
    eg: /restful/fight/testuser/Ellymoo/Forest

  • /restful/users - to get a list of all users
    eg: /restful/users

  • /restful/users/{userName} - to get one user by user name
    eg: /restful/users/testuser

  • /restful/characters/{userName} - to get a list of characters belonging to a user, by user name
    eg: /restful/characters/testuser

  • /restful/characters/{userName}/{charName} - to get one character belonging to a user, by user name and character name
    eg: /restful/characters/testuser/Peticel

  • /restful/locations - to get a list of all locations
    eg: /restful/locations

  • /restful/monsters/{locationName} - to get a list of all monsters at a location, by location name
    eg: /restful/monsters/Ruins

  • /restful/types - to get a list of all character types (classes)
    eg: /restful/types

  • /restful/weapons - to get a list of all weapons
    eg: /restful/weapons
